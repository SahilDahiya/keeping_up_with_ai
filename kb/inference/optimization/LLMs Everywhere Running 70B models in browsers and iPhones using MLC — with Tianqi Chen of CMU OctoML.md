# LLMs Everywhere: Running 70B models in browsers and iPhones using MLC — with Tianqi Chen of CMU / OctoML

*We have just announced our first set of speakers at  AI Engineer Summit! Sign up for the livestream or email [email protected] if you’d like to support.*

We are facing a [massive GPU crunch](https://twitter.com/tqchenml/status/1689397314576662528). As both startups and VC’s hoard Nvidia GPUs like countries count nuclear stockpiles, [tweets about GPU shortages](https://twitter.com/abacaj/status/1687946803377782784?s=20) have become increasingly common.

*But what if we could run LLMs with AMD cards, or without a GPU at all?* There’s just one weird trick: **compilation.** And there’s one person uniquely qualified to do it.

We had the pleasure to sit down with [Tianqi Chen](https://www.linkedin.com/in/tianqi-chen-679a9856/), who’s an Assistant Professor at CMU, where he both teaches the [MLC course](https://mlc.ai/summer22/) and runs the MLC group. You might also know him as the creator of [XGBoost](https://www.nvidia.com/en-us/glossary/data-science/xgboost/), [Apache TVM](https://tvm.apache.org/), and [MXNet](https://mxnet.apache.org/versions/1.9.1/), as well as the co-founder of [OctoML](http://octoml.ai).

The MLC (short for [Machine Learning Compilation](https://mlc.ai/)) group has released a lot of interesting projects:

- [MLC Chat](https://testflight.apple.com/join/57zd7oxa):
- [Web LLM](https://webllm.mlc.ai/):
- [MLC LLM:](https://github.com/mlc-ai/mlc-llm)

The MLC group has [just announced](https://twitter.com/junrushao/status/1689321439625883648?s=20) new support for AMD cards; we previously talked about the shortcomings of ROCm, but using MLC you can get performance very close to the NVIDIA’s counterparts. This is great news for founders and builders, as AMD cards are more readily available. Here are their latest results on AMD’s 7900s vs some of top NVIDIA consumer cards.

![Image Image](https://substackcdn.com/image/fetch/$s_!C10j!,w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2Fe4a3ebe1-930c-48b9-accc-ea4599b04cb2_1798x844.png)

If you just can’t get a GPU at all, MLC LLM also supports ARM and x86 CPU architectures as targets by leveraging LLVM. While speed performance isn’t comparable, it allows for non-time-sensitive inference to be run on commodity hardware.

We also enjoyed getting a peek into TQ’s process, which [involves a lot of sketching](https://twitter.com/FanaHOVA/status/1689680222726283264):

![Image Image](https://substackcdn.com/image/fetch/$s_!j5io!,w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2F82d6660d-dfaa-44ca-82b6-4dd0cb059d7b_4032x3024.jpeg)

With all the other work going on in this space with projects like [ggml](https://github.com/ggerganov/ggml) and [Ollama](https://ollama.ai/), we’re excited to see GPUs becoming less and less of an issue to get models in the hands of more people, and innovative software solutions to hardware problems!
